Desafio 1 - Tipagem de objetos "Employee"

No primeiro desafio, temos um código em JavaScript (representado abaixo) e o objetivo é representar ele em um arquivo TypeScript.

// Como podemos rodar isso em um arquivo .ts sem causar erros? 
let employee = { };
employee.code = 10;
employee.name = "John";

Solução 1: Interfaces

A primeira solução possível é definir uma interface “Employee”, contendo as propriedades “code”, do tipo “number e “name” do tipo “string”.

interface Employee {
    code: number;
    name: string;
}

let employee: Employee = {
    code: 10,
    name: "John"
}

let employee2 = { } as Employee;
employee2.code = 10;
employee2.name = "John";

Solução 2: Inline typing

Outra solução é utilizar uma tipagem em linha, que é descrever o tipo no momento em que a variável é declarada.

let employee:  { code: number, name: string } =  {
    code: 10,
    name: "John"
}

Conclusão

Dentre as duas soluções, a primeira seria a mais escalável, pois não seria necessário repetir o código de descrição de tipo toda vez que fosse declarar uma variável “Employee”. Entretanto, caso o tipo seja usado uma única vez, não tem problema usar a segunda solução, entretanto, caso seja necessário usar o tipo em mais de um lguar, recomenda-se mudar para a primeira solução.
